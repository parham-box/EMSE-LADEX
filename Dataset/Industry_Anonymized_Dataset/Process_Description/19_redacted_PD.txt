1. Start the procedure.

2. Determine whether a data backup is required by checking the date and time of the last backup.

3. Refer to [DOCUMENT_A].

4. Verify the current alarm conditions on the network element.

5. Verify that the replacement [COMPONENT_ID] is the same as the current [COMPONENT_ID].

6. Switch traffic away from this optical link by performing [PROCEDURE].

7. Refer to [DOCUMENT_B].

8. Ensure that the [INDICATOR_LED] on the module faceplate is off.

Refer to [DATA_SOURCE_A]  
Refer to [DATA_SOURCE_B] 
Refer to [DATA_SOURCE_C]   

9. Ensure that labels are applied to both ends of all cables and patchcords.

10. Disconnect all [TYPE] connectors from the module to be replaced.

11. Put protective caps on the ends of the patchcords.

12. Push outwards on both pull tabs simultaneously and pull the module out of the slot.

13. Gently pull the module until it is fully removed.

14. Install the replacement module into the corresponding slot and press the latches to close.

15. Refer to [DOCUMENT_C].

16. You can retrieve the inventory data for the module you inserted.

If you want to perform the procedure using then go to [Use Command Line Interface]

If you want to perform the procedure using then go to [Use Graphical User Interface]  

CLI INTERFACE STEPS
17. Enter [COMMAND] to show slot inventory details.
[EXAMPLE]

GUI INTERFACE STEPS
18. From the [MENU_A], select the module you want to retrieve the inventory for.

19. Clean all disconnected fiber connectors and reconnect them to the appropriate ports on the module.

20. For instructions on inspecting and cleaning optical fiber, refer to [DOCUMENT_D].

21. Wait for the module to start.

22. Retrieve all the active alarms and alarm details.

 Which interface will be used to retrieve the active alarms and alarm details?
 If Use Command Line Interface then go to CLI ALARMS STEPS
 If Use Graphical User Interface then go to GUI ALARMS STEPS

CLI ALARMS STEPS
23. Enter [COMMAND] to show active alarms.
[EXAMPLE]

GUI ALARMS STEPS
24. Retrieve the data for the active alarm object; refer to [DOCUMENT_E].

25. Is the '[ALARM_A]' alarm active?
- If Yes: 26. Wait until the alarm clears before continuing.
- If No: proceed to step 27.

27. Is the '[ALARM_B]' alarm active?
- If Yes: 28. A problem with provisioning the module has been detected.
- If No: proceed to step 29.

29. Undo any traffic switches invoked.

30. Update the user label with the current shelf and slot.

31. Ensure there are no active alarms against the newly installed module(s).

32. You have completed this procedure.
